SCREENSHOT SPECIFICATIONS
=========================

File Formats:
- PNG (recommended for screenshots)
- JPG (acceptable for photos)
- SVG (for graphics)

Resolution:
- Desktop: 1920x1080 (Full HD)
- Laptop: 1366x768
- Tablet: 1024x768
- Mobile: 375x667 (iPhone) / 390x844 (Android)

File Naming Convention:
- kebab-case naming
- Descriptive names
- Include device/resolution if needed
Example: landing-page-desktop.png

CONTENT GUIDELINES
==================

What to Include:
- Beautiful, professional layouts
- Clean, uncluttered interfaces
- Real example data when possible
- Consistent branding
- Accessibility compliance
- Mobile responsiveness

What to Avoid:

- Placeholder text ("Lorem ipsum")
- Inconsistent styling
- Pixelated images
- Debug mode enabled
- Developer tools visible

PROCESSING
==========

Tools:
- Chrome DevTools
- Browser Screenshot extensions
- Image editing software (optional)
- Batch compression tools

Processing Steps:
1. Take screenshot at full resolution
2. Crop to appropriate size
3. Compress if needed (maintain quality)
4. Verify clarity and readability
5. Check file size (optimize if too large)
6. Rename with descriptive filename
7. Add to appropriate subfolder

QUALITY STANDARDS
=================

Resolution: Minimum 1280x720 for desktop
File Size: Keep under 1MB when possible
Format: PNG for screenshots, JPG for photos
Compression: Minimal loss, maintain clarity
Colors: Accurate color representation
Text: All text must be readable
Contrast: Sufficient contrast for visibility
